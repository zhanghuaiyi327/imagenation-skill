# 增量内容合并提示词

## 目标
当用户提供新信息时，智能分析增量内容，将其合并到现有伴侣档案中，实现档案的持续进化和优化。

## 使用场景
在伴侣档案生成后，用户提供新信息或进行纠正时使用。

## 合并流程

### 1. 新信息分析

#### 1.1 信息类型识别

**信息类型分类：**
- **补充信息**：新增的前任分析、具体要求等
- **纠正信息**：修正现有档案中的错误或不准确内容
- **更新信息**：反映认知变化或新发现的信息
- **细化信息**：对现有内容的详细补充

**识别提示：**
```
分析新信息类型：

1. 内容分析：
   - 信息主题：[识别]
   - 信息性质：[补充/纠正/更新/细化]
   - 信息重要性：[高/中/低]

2. 影响范围分析：
   - 影响的档案部分：[列出]
   - 影响的分析维度：[列出]
   - 影响的兼容性评估：[分析]

3. 处理优先级：
   - 紧急程度：[高/中/低]
   - 处理复杂度：[高/中/低]
   - 预期影响：[大/中/小]
```

#### 1.2 信息质量评估

**评估维度：**
- 信息具体程度
- 信息可信度
- 信息一致性
- 信息完整性

**评估提示：**
```
评估新信息质量：

1. 具体程度评估：
   - 是否有具体例子
   - 是否有详细描述
   - 是否有明确结论

2. 可信度评估：
   - 信息来源可靠性
   - 信息逻辑合理性
   - 信息情感中立性

3. 一致性评估：
   - 与现有信息的一致性
   - 内部逻辑一致性
   - 时间线一致性

4. 完整性评估：
   - 信息是否完整
   - 是否有缺失部分
   - 是否需要进一步澄清
```

### 2. 现有档案分析

#### 2.1 档案结构理解

**分析内容：**
- 当前档案版本
- 档案各部分内容
- 分析逻辑和依据
- 兼容性评估基础

**分析提示：**
```

1. 档案概览：
   - 当前版本：{version}
   - 生成时间：{timestamp}
   - 数据基础：{data_basis}

2. 内容分析：

   - 价值观体系部分：[分析]
   - 生活方式部分：[分析]
   - 兼容性分析部分：[分析]

3. 逻辑理解：
   - 分析推理过程：[理解]
   - 评估依据：[识别]
   - 结论基础：[分析]
```

#### 2.2 档案脆弱点识别

**脆弱点类型：**
- 数据不足部分
- 逻辑薄弱环节
- 矛盾信息点
- 需要更新内容

**识别提示：**
```
识别档案脆弱点：

1. 数据脆弱点：
   - 数据不足的领域：[列出]

   - 可能过时的信息：[识别]
   - 认知变化影响的部分：[识别]
```

### 3. 合并策略制定

#### 3.1 合并方式选择

**合并方式：**
- **补充合并**：新增信息补充到现有内容
- **替换合并**：用新信息替换旧信息
- **调整合并**：基于新信息调整现有内容
- **重构合并**：重新分析构建相关内容

**选择提示：**
```
选择合并方式：

1. 基于信息类型：
   - 补充信息 → 补充合并
   - 纠正信息 → 替换合并
   - 更新信息 → 调整合并
   - 重大变化 → 重构合并

2. 基于影响范围：
   - 局部影响 → 补充/替换合并
   - 中等影响 → 调整合并
   - 重大影响 → 重构合并

3. 基于信息质量：
   - 高质量信息 → 替换/重构合并
   - 中等质量信息 → 调整合并
   - 低质量信息 → 补充合并（需验证）
```

#### 3.2 合并优先级确定

**优先级因素：**
- 信息重要性
- 影响范围大小
- 合并复杂度
- 用户紧急程度

**确定提示：**
```
确定合并优先级：

1. 重要性评估：
   - 核心性格特征：高优先级
   - 关键价值观：高优先级
   - 重要生活方式：中高优先级
   - 次要细节：低优先级

2. 影响评估：
   - 影响兼容性评估：高优先级
   - 影响关系建议：高优先级
   - 影响档案完整性：中优先级
   - 影响次要内容：低优先级

3. 复杂度评估：
   - 简单合并：立即处理
   - 中等复杂度：计划处理
   - 高复杂度：分阶段处理
```

### 4. 具体合并操作

#### 4.1 性格特征合并

**合并规则：**
- 新增性格特征：补充到相应维度
- 纠正性格特征：替换错误描述
- 细化性格特征：丰富现有描述
- 矛盾性格特征：重新分析验证

**操作提示：**
```
合并性格特征：

1. 大五人格合并：
   - 检查新信息对各个维度的影响
   - 更新维度评分和描述
   - 调整性格优势分析

2. 关系特征合并：
   - 更新情感表达风格
   - 调整亲密需求模式
   - 修正边界意识描述
   - 完善支持行为模式

3. 特征整合：
   - 确保特征间逻辑一致
   - 更新性格档案摘要
   - 调整成长发展建议
```

#### 4.2 价值观体系合并

**合并规则：**
- 新增价值观：补充到体系
- 纠正价值观：修正错误认知
- 更新价值观：反映认知变化
- 细化价值观：丰富具体体现

**操作提示：**
```
合并价值观体系：

1. 核心价值观合并：
   - 更新家庭观念描述
   - 调整事业追求内容
   - 完善个人成长理念
   - 修正社会责任认知

2. 关系价值观合并：
   - 更新忠诚信任观念
   - 调整沟通表达理念
   - 完善个人空间平衡
   - 修正冲突解决哲学

3. 价值观整合：
   - 确保价值观一致性
   - 更新价值观体现描述
   - 调整关系行为准则
```

#### 4.3 生活方式合并

**合并规则：**
- 新增生活习惯：补充到描述
- 纠正生活模式：修正错误信息
- 更新生活偏好：反映变化
- 细化生活细节：丰富描述

**操作提示：**
```
合并生活方式：

1. 日常生活合并：
   - 更新作息规律描述
   - 调整生活习惯内容
   - 完善家务管理理念
   - 修正财务管理观念

2. 社交休闲合并：
   - 更新社交风格描述
   - 调整兴趣爱好内容
   - 完善休闲活动偏好
   - 修正假期规划理念

3. 生活方式整合：
   - 确保生活模式一致性
   - 更新生活管理特点
   - 调整生活协调建议
```

#### 4.4 兼容性分析更新

**更新规则：**
- 基于新信息重新评估兼容性
- 更新匹配度评分
- 调整冲突点识别
- 修正发展建议

**操作提示：**
```
更新兼容性分析：

1. 重新评估：
   - 基于更新后的性格特征
   - 基于更新后的价值观
   - 基于更新后的生活方式

2. 评分更新：
   - 重新计算各维度评分
   - 更新综合评分
   - 调整兼容性等级

3. 分析更新：
   - 更新匹配点分析
   - 调整冲突点识别
   - 修正风险预警

4. 建议更新：
   - 基于新分析更新关系建议
   - 调整发展策略
   - 更新成长计划
```

### 5. 合并质量保证

#### 5.1 一致性检查

**检查内容：**
- 各部分逻辑一致性
- 信息前后一致性
- 分析推理一致性
- 结论依据一致性

**检查提示：**
```
进行一致性检查：

1. 逻辑检查：
   - 性格特征与行为描述是否一致
   - 价值观与生活选择是否一致
   - 分析结论与依据是否一致

2. 信息检查：
   - 新旧信息是否协调
   - 不同部分信息是否一致
   - 具体描述与总体结论是否一致

3. 推理检查：
   - 分析过程是否逻辑连贯
   - 结论是否得到充分支持
   - 假设是否合理明确
```

#### 5.2 完整性检查

**检查内容：**
- 所有新信息都处理
- 所有影响都考虑
- 所有更新都完成
- 所有检查都通过

**检查提示：**
```
进行完整性检查：

1. 处理完整性：
   - 所有新信息是否都分析
   - 所有影响是否都评估
   - 所有更新是否都实施

2. 内容完整性：
   - 档案各部分是否都更新
   - 分析维度是否都覆盖
   - 结论是否都调整

3. 质量完整性：
   - 质量检查是否都通过
   - 错误处理是否都完成
   - 验证步骤是否都执行
```

## 合并工作流程

### 标准工作流程：
```
1. 接收新信息
2. 分析信息类型和质量
3. 分析现有档案状态
4. 制定合并策略
5. 执行具体合并操作
6. 进行质量保证检查
7. 生成合并报告
8. 更新档案版本
```

### 紧急工作流程（重大纠正）：
```
1. 立即分析纠正信息
2. 识别受影响的核心内容
3. 优先处理关键修正
4. 快速更新档案
5. 通知用户关键变化
6. 后续完善其他部分
```

## 版本管理规则

### 版本命名规则：
- 主版本：重大重构或核心变化（v1.0 → v2.0）
- 次版本：重要更新或功能增加（v1.0 → v1.1）
- 修订版本：错误修正或小调整（v1.0.0 → v1.0.1）

### 版本记录要求：
- 记录每次合并的版本变化
- 记录变更内容和原因
- 记录合并时间和操作者
- 记录质量检查结果
