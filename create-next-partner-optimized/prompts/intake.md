# 基础信息收集提示词

## 目标
引导用户提供创建理想伴侣Skill所需的所有基础信息，包括前任分析、经验教训和对下一任的具体要求。

## 使用场景
当用户启动`/create-next-partner`命令或表达创建理想伴侣的意图时使用。

## 核心问题序列

### 第一阶段：伴侣代号和基本信息

**问题1：伴侣代号**
```
请为你的理想伴侣起一个代号（用于文件命名和后续引用）：
- 建议使用英文或拼音，避免特殊字符
- 例如：ideal_partner_2024、dream_companion、perfect_match
```

**问题2：前任姓名（可选）**
```
前任的姓名或称呼是什么？（可选，用于数据采集时的参考）
```

**问题3：关系持续时间**
```
你们的关系持续了多长时间？
- 格式：如"2年"、"6个月"、"3年半"
- 如果有多段关系，请描述最相关的一段
```

**问题4：分手主要原因**
```
导致分手的主要原因是什么？
- 请具体描述，避免模糊表述
- 例如：价值观冲突、沟通问题、信任危机、生活方式不合等
```

### 第二阶段：关键经验教训

**引导语：**
```
从这段关系中，你学到了哪些重要的经验教训？
这些教训将帮助我们避免重复错误，构建更健康的关系。

请至少提供3个经验教训，每个教训应该：
1. 具体明确
2. 有实际例子支持
3. 包含对未来的指导意义

输入格式：每行一个教训，输入"完成"结束
```

**示例回答：**
```
1. 沟通要及时：不要积累问题，小问题及时沟通可以避免大冲突
2. 尊重个人空间：每个人都需要独立的时间和空间，过度依赖会造成压力
3. 价值观要一致：在重要问题上（如家庭观、金钱观）需要有共识
完成
```

### 第三阶段：前任优点分析

**引导语：**
```
现在我们来分析前任的优点和长处。
这些优点是你希望在下一任中继承的特质。

请至少提供3个具体优点，每个优点需要：
1. 具体特质描述
2. 实际例子说明
3. 这个优点对关系的影响

格式：
优点1:
  特质：[具体特质]
  例子：[具体例子]
  影响：[对关系的影响]
```

**示例回答：**
```
优点1:
  特质：体贴关心
  例子：生病时会主动照顾，记得重要纪念日
  影响：让人感受到被爱和被重视，增强关系安全感

优点2:
  特质：幽默风趣
  例子：能调节气氛，让日常相处轻松愉快
  影响：减少关系中的压力，增加生活乐趣

优点3:
  特质：有责任心
  例子：对承诺认真，工作生活都有规划
  影响：关系稳定可预测，有安全感
```

### 第四阶段：前任缺点分析

**引导语：**
```
现在分析前任的缺点和问题。
这些是需要避免或改进的地方。

请至少提供2个具体缺点，每个缺点需要：
1. 具体问题描述
2. 实际影响说明
3. 是否必须避免（是/否）

格式：
缺点1:
  问题：[具体问题]
  影响：[具体影响]
  必须避免：[是/否]
```

**示例回答：**
```
缺点1:
  问题：情绪不稳定
  影响：经常因为小事发脾气，情绪波动大，难以预测
  必须避免：是

缺点2:
  问题：缺乏安全感
  影响：需要频繁确认关系，造成双方压力
  必须避免：是
```

### 第五阶段：下一任要求

#### 5.1 必须拥有的特质（非妥协项）

**引导语：**
```
对下一任，有哪些特质是必须拥有的？
这些是非妥协项，缺少任何一项都无法接受。

请至少提供3个必须特质，每个特质需要：
1. 具体特质描述
2. 为什么这个特质很重要
3. 优先级（高/中/低）

格式：
必须特质1:
  特质：[具体特质]
  重要性：[为什么重要]
  优先级：[高/中/低]
```

**示例回答：**
```
必须特质1:
  特质：情绪稳定
  重要性：希望关系稳定可预测，避免情绪波动带来的压力
  优先级：高

必须特质2:
  特质：善于沟通
  重要性：能有效解决问题，避免误解积累
  优先级：高

必须特质3:
  特质：尊重个人空间
  重要性：需要独立性和信任，避免过度依赖
  优先级：高
```

#### 5.2 希望拥有的特质（加分项）

**引导语：**
```
有哪些特质是希望拥有的？
这些是加分项，但不是必须的。

格式：每行一个特质，输入"完成"结束
```

**示例回答：**
```
喜欢旅行
有幽默感
有共同兴趣爱好
完成
```

#### 5.3 绝对不能接受的特质（底线）

**引导语：**
```
有哪些特质是绝对不能接受的？
这些是关系底线，一旦出现就必须结束关系。

每个底线特质需要：
1. 具体特质描述
2. 为什么不能接受

格式：
底线特质1:
  特质：[具体特质]
  原因：[为什么不能接受]
```

**示例回答：**
```
底线特质1:
  特质：控制欲强
  原因：需要自由和尊重，控制会破坏关系平衡

底线特质2:
  特质：不诚实
  原因：信任是关系的基础，不诚实会彻底破坏信任
```

#### 5.4 生活方式偏好

**引导语：**
```
请描述你对生活方式的具体偏好：

1. 工作生活平衡偏好：
2. 社交风格偏好：
3. 未来3-5年生活规划：
4. 兴趣爱好偏好：
```

**示例回答：**
```
1. 工作生活平衡偏好：希望工作和生活有明确界限，周末尽量不工作
2. 社交风格偏好：喜欢小范围深度交流，不喜欢大型社交活动
3. 未来3-5年生活规划：稳定工作，考虑组建家庭，每年旅行1-2次
4. 兴趣爱好偏好：音乐、阅读、户外运动、美食探索
```

## 数据验证规则

### 必填项检查
1. 伴侣代号：不能为空，只能包含字母、数字、下划线
2. 关系持续时间：不能为空
3. 分手原因：不能为空
4. 经验教训：至少3个
5. 前任优点：至少3个，每个需要特质、例子、影响
6. 前任缺点：至少2个，每个需要问题、影响、是否避免
7. 必须特质：至少3个，每个需要特质、重要性、优先级

### 格式验证
1. 所有文本字段：去除首尾空格，检查是否为空
2. 优先级字段：只能是"高"、"中"、"低"
3. 是否避免字段：只能是"是"或"否"

### 逻辑验证
1. 必须特质和底线特质不能有冲突
2. 前任优点和缺点不能有逻辑矛盾
3. 经验教训应该与分手原因相关

## 错误处理

### 数据不完整
```
⚠️ 数据不完整，请补充以下信息：
- [缺少的信息项]
请重新输入或补充相关信息。
```

### 格式错误
```
⚠️ 格式错误：[具体错误]
请按照要求格式重新输入。
```

### 逻辑矛盾
```
⚠️ 发现逻辑矛盾：[具体矛盾]
请检查并修正相关信息。
```
