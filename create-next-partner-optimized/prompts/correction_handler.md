# 纠正处理提示词

## 目标
当用户表达"不对"、"他不会这样"、"他应该是"等纠正意图时，准确识别纠正内容，智能处理纠正请求，更新伴侣档案。

## 使用场景
用户对生成的伴侣档案内容提出纠正时使用。

## 纠正识别

### 1. 纠正意图识别

#### 1.1 纠正表达模式

**常见纠正表达：**
- 直接否定："不对"、"错了"、"不是这样"
- 对比纠正："他不会这样，他会..."
- 补充纠正："还有..."、"另外..."
- 细化纠正："更准确地说..."、"具体是..."
- 情感纠正："我不喜欢这样"、"我希望..."

**识别提示：**
```
识别纠正意图：

1. 语言模式分析：
   - 否定词出现：[识别]
   - 对比结构出现：[识别]
   - 补充说明出现：[识别]
   - 情感表达出现：[识别]

2. 纠正强度评估：
   - 轻微纠正：细节调整
   - 中等纠正：部分修正
   - 重大纠正：核心内容改变

3. 纠正类型判断：
   - 事实纠正：错误信息修正
   - 观点纠正：不同看法表达
   - 偏好纠正：个人偏好说明
   - 情感纠正：情感反应表达
```

#### 1.2 纠正内容提取

**提取维度：**
- 纠正的具体内容
- 纠正的替代描述
- 纠正的原因说明
- 纠正的情感背景

**提取提示：**
```
提取纠正内容：

1. 内容提取：
   - 被纠正的内容：[提取]
   - 正确的描述：[提取]
   - 纠正原因：[提取]
   - 情感背景：[分析]

2. 内容验证：
   - 提取是否完整：[检查]
   - 内容是否清晰：[检查]
   - 逻辑是否合理：[检查]
   - 情感是否理解：[分析]

3. 内容分类：
   - 属于哪个档案部分：[分类]
   - 影响哪些分析维度：[分析]
   - 需要哪些调整：[识别]
```

### 2. 纠正影响分析

#### 2.1 影响范围分析

**分析维度：**
- 直接影响的内容
- 间接影响的分析
- 兼容性评估影响
- 关系建议影响

**分析提示：**
```
分析纠正影响：

1. 直接影响分析：
   - 具体影响的档案内容：[列出]
   - 需要修改的描述：[识别]
   - 需要更新的分析：[识别]

2. 间接影响分析：
   - 影响的相关分析：[分析]
   - 影响的逻辑推理：[检查]
   - 影响的结论评估：[评估]

3. 系统性影响：
   - 对性格特征的影响：[分析]
   - 对价值观的影响：[分析]
   - 对生活方式的影响：[分析]
   - 对兼容性的影响：[分析]
```

#### 2.2 纠正严重性评估

**评估标准：**
- 核心内容纠正：高严重性
- 重要细节纠正：中严重性
- 次要描述纠正：低严重性
- 个人偏好纠正：个性化处理

**评估提示：**
```
评估纠正严重性：

1. 内容重要性评估：
   - 核心性格特征：高重要性
   - 关键价值观：高重要性
   - 重要行为模式：中高重要性
   - 次要生活细节：低重要性

2. 纠正准确性评估：
   - 明显错误：高严重性
   - 不准确描述：中严重性
   - 表达偏差：低严重性
   - 个人观点：个性化

3. 影响范围评估：
   - 广泛影响：高严重性
   - 局部影响：中严重性
   - 微小影响：低严重性
   - 无影响：忽略
```

### 3. 纠正处理策略

#### 3.1 处理方式选择

**处理方式：**
- **直接修正**：替换错误内容
- **补充说明**：增加额外信息
- **调整描述**：修改表达方式
- **重新分析**：基于新信息重新分析
- **个性化调整**：根据用户偏好调整

**选择提示：**
```
选择处理方式：

1. 基于纠正类型：
   - 事实错误 → 直接修正
   - 信息不全 → 补充说明
   - 表达不准 → 调整描述
   - 重大变化 → 重新分析
   - 个人偏好 → 个性化调整

2. 基于严重性：
   - 高严重性 → 重新分析
   - 中严重性 → 调整描述
   - 低严重性 → 补充说明
   - 个性化 → 个性化调整

3. 基于影响范围：
   - 广泛影响 → 重新分析
   - 局部影响 → 调整描述
   - 微小影响 → 直接修正
```

#### 3.2 处理优先级确定

**优先级因素：**
- 纠正严重性
- 影响范围大小
- 用户紧急程度
- 处理复杂度

**确定提示：**
```
确定处理优先级：

1. 紧急程度评估：
   - 核心内容错误：立即处理
   - 重要不准确：高优先级
   - 次要问题：中优先级
   - 微小偏差：低优先级

2. 用户需求评估：
   - 用户明确要求立即处理：高优先级
   - 用户表达不满：高优先级
   - 用户建议性纠正：中优先级
   - 用户补充信息：按需处理

3. 系统影响评估：
   - 影响后续分析：高优先级
   - 影响用户体验：高优先级
   - 影响档案质量：中优先级
   - 影响次要内容：低优先级
```

## 工作流程模板

### 标准纠正处理流程：
```
1. 接收用户纠正表达
2. 识别纠正意图和内容
3. 分析纠正影响和严重性
4. 选择处理方式和优先级
5. 执行具体纠正操作
6. 进行质量保证检查
7. 创建纠正记录
8. 沟通处理结果
9. 学习改进
```

### 紧急纠正处理流程（重大错误）：
```
1. 立即识别重大纠正
2. 快速分析核心影响
3. 优先处理关键错误
4. 立即通知用户处理
5. 实施紧急修正
6. 后续完善其他部分
7. 全面质量检查
8. 系统改进学习
```
