# 前任性格分析提示词

## 目标
基于用户提供的前任信息（优点、缺点、经验教训）和自动化采集的数据，深入分析前任的性格特征、行为模式和关系动态。

## 使用场景
在收集完基础信息和原材料后，用于系统分析前任的性格特征。

## 分析框架

### 1. 沟通风格分析

#### 分析维度：
1. **表达方式**
   - 直接 vs 间接
   - 主动 vs 被动
   - 情感表达 vs 理性表达

2. **倾听能力**
   - 是否认真倾听
   - 是否理解对方观点
   - 是否给予反馈

3. **冲突处理**
   - 面对冲突的反应
   - 解决问题的方式
   - 妥协和协商能力

#### 数据来源：
- 用户描述的前任优点/缺点
- 聊天记录（如果提供）
- 邮件/文档内容
- 用户提供的具体例子

#### 分析提示：
```
基于以下信息分析前任的沟通风格：

1. 从用户描述中提取沟通相关特质：
   - 优点中提到的沟通相关特质：[列出]
   - 缺点中提到的沟通问题：[列出]
   - 经验教训中关于沟通的教训：[列出]

2. 从自动化数据中分析沟通模式：
   - 消息频率和长度
   - 情感词汇使用频率
   - 问题解决方式
   - 表达清晰度

3. 综合评估沟通风格：
   - 主要沟通风格：[描述]
   - 沟通优势：[列出]
   - 沟通弱点：[列出]
   - 对关系的影响：[分析]
```

### 2. 情绪管理分析

#### 分析维度：
1. **情绪稳定性**
   - 日常情绪波动
   - 压力下的情绪反应
   - 情绪恢复能力

2. **情感表达**
   - 表达情感的方式
   - 情感深度和真实性
   - 情感需求表达

3. **同理心**
   - 理解他人情感的能力
   - 情感支持提供
   - 情感共鸣程度

#### 分析提示：
```
分析前任的情绪管理能力：

1. 情绪相关特质提取：
   - 情绪稳定性相关描述：[从优点/缺点中提取]
   - 情感表达方式：[从例子中分析]
   - 同理心表现：[从具体事例中分析]

2. 情绪模式识别：
   - 常见情绪触发点
   - 情绪表达频率和强度
   - 情绪调节策略

3. 情绪管理评估：
   - 情绪稳定性评分：1-10分
   - 情感表达健康度：1-10分
   - 同理心水平：1-10分
   - 需要改进的情绪管理方面：[列出]
```

### 3. 价值观体系分析

#### 分析维度：
1. **核心价值观**
   - 家庭观念
   - 事业追求
   - 个人成长
   - 社会责任

2. **关系价值观**
   - 忠诚和信任
   - 沟通和表达
   - 个人空间和亲密
   - 冲突解决理念

3. **生活哲学**
   - 对生活的态度
   - 对未来的规划
   - 对困难的应对方式

#### 分析提示：
```
深入分析前任的价值观体系：

1. 价值观提取：
   - 从经验教训中提取价值观线索
   - 从生活方式偏好中分析价值观
   - 从具体行为中推断价值观

2. 价值观一致性评估：
   - 言行一致性
   - 长期价值观稳定性
   - 价值观冲突点

3. 价值观对关系的影响：
   - 价值观匹配度分析
   - 价值观冲突的具体表现
   - 价值观调整的可能性
```

### 4. 行为模式分析

#### 分析维度：
1. **日常行为模式**
   - 作息规律
   - 生活习惯
   - 社交行为

2. **关系行为模式**
   - 亲密行为
   - 支持行为
   - 冲突行为

3. **压力应对模式**
   - 压力下的行为变化
   - 应对策略
   - 恢复能力

#### 分析提示：
```
识别前任的行为模式：

1. 行为模式提取：
   - 从具体例子中提取行为模式
   - 从自动化数据中分析行为规律
   - 从用户描述中识别行为特征

2. 行为模式分类：
   - 健康行为模式：[列出]
   - 问题行为模式：[列出]
   - 中性行为模式：[列出]

3. 行为模式影响：
   - 对关系的积极影响
   - 对关系的消极影响
   - 行为模式改变的可能性
```

### 5. 冲突处理分析

#### 分析维度：
1. **冲突触发点**
   - 常见冲突原因
   - 冲突频率
   - 冲突强度

2. **冲突应对策略**
   - 面对冲突的态度
   - 解决冲突的方法
   - 妥协和让步程度

3. **冲突后处理**
   - 道歉和修复
   - 学习和改进
   - 关系恢复

#### 分析提示：
```
分析前任的冲突处理方式：

1. 冲突相关信息提取：
   - 分手原因中的冲突因素
   - 缺点中提到的冲突问题
   - 经验教训中的冲突相关教训

2. 冲突处理模式识别：
   - 主要冲突处理风格
   - 冲突解决有效性
   - 冲突后关系修复能力

3. 冲突处理评估：
   - 冲突处理健康度：1-10分
   - 需要改进的冲突处理方面
   - 建议的冲突处理策略
```

## 数据整合规则

### 1. 多源数据整合
- 用户手动输入信息优先
- 自动化采集数据作为补充
- 矛盾信息需要验证和澄清

### 2. 信息权重分配
- 具体例子权重高于一般描述
- 重复出现的信息权重更高
- 情感强烈的描述需要特别关注

### 3. 分析深度控制
- 核心问题深入分析
- 次要问题简要分析
- 不确定信息标注存疑

## 输出格式要求

### 1. 结构化输出
- 使用清晰的标题和子标题
- 列表项使用项目符号
- 评分使用1-10分制

### 2. 语言风格
- 客观中立，避免主观判断
- 具体明确，避免模糊表述
- 建设性，提供改进建议

### 3. 长度控制
- 总体报告：800-1200字
- 每个分析部分：150-250字
- 关键洞察：50-100字
